Every skill in the gm chain

thirteen skills, one chair, no skipping
cushion-certified · 2026
tl;dr
Skills are how the agent moves between phases of the chair. The state machine — PLAN → EXECUTE → EMIT → VERIFY → UPDATE-DOCS → COMPLETE — is enforced by skill invocations, one per transition. There is no auto-chaining. Each step is an explicit Skill(...) call. Below: every skill in the standard chain, plus the supporting skills (browser, ssh, code-search, governance, pages, create-lang-plugin) and the memorize sub-agent.

Glootie says:"There is no skip button. There is no ‘just this once.’ The arrows are arrows. The cushion is the cushion. Sit."

gm

root orchestrator always first
roleState-machine entrypoint. Loads policy: recall, memorize, autonomy, lawful-downgrade, fix-on-sight, browser-witness. Immediately invokes planning.
fires whenEvery user message. The hook layer writes .gm/needs-gm on every prompt — the next non-Skill tool call is denied until Skill(gm) clears it.
forbidsSkipping invocation. Loading skills via the Agent tool. Direct phase invocation without going through gm → planning.
why it existsWithout a forced first action, agents drift into "let me just check…" patterns that bypass the state machine. gm is the cushion you sit on before you do anything else.

planning

orient · cover · PRD PLAN
roleORIENT first (a parallel pack of recalls and codesearches against the request's nouns), then enumerate the covering family, then write the PRD. The PRD is non-negotiable for any task beyond a single-line edit.
fires whenAfter gm. Re-fires on any new unknown surfaced in any later phase (regression).
forbidsnaming an unknown before ORIENT. single-subset triage when other witnessable subsets exist (distributed refusal). Skipping the PRD on multi-step work. Items without acceptance criteria. Reading from a stale codeinsight or search cache instead of a freshly-synced index.
why it existsIf the cushion is the discipline, ORIENT is sitting down and the PRD is the seat-belt. Skipping ORIENT or skipping the PRD pays the same cost in fresh probes a turn later — plus the duplicated discovery, plus the risk of disagreeing with the agent's own prior witness.

gm-execute

execution contract for every phase EXECUTE (and shared)
roleResolve every mutable from PLAN by witnessed execution. Mutables become KNOWN only when ΔS=0 (witnessed equals expected), λ≥2 (two independent paths agree), ε intact (adjacent invariants hold), Coverage≥0.70 (enough corpus inspected).
fires whenAfter PLAN exits. Also: about to run anything in any phase — load this skill first as the execution contract.
forbidsunwitnessed claims. Bash(node|npm|npx|bun). Mocks, stubs, fallbacks, swallowed errors. Editing before witnessing. Reimplementing a module you should have imported.
why it existsPlausibility is not authorization. gm-execute downgrades every prior to weak_prior until a witnessed probe upgrades it.

gm-emit

file writer + post-emit verifier EMIT
rolePre-emit debug check (legitimacy gate per governance: earned specificity, repair legality, lawful downgrade preferred over forced closure). Write files. Post-emit verify from disk: read what you wrote, re-witness invariants. New unknown surfaced → snake back to planning.
fires whenEXECUTE has driven all mutables to KNOWN.
forbidsFive refused collapses (governance §legitimacy). Writing without a post-emit verify. Treating compile-success as behavior-witness.
why it existsThe moment of writing is the moment of risk. Most regressions land here. gm-emit insists on reading back what you just wrote.

gm-complete

end-to-end verify + git enforcement VERIFY
roleRun the single test.js at project root. Walk the 8-case governance stress suite (M1/F1/C1/H1/S1/B1/A1/D1). Check git: nothing uncommitted, nothing unpushed, CI green for the pushed HEAD. New unknown → snake back to PLAN.
fires whenEMIT completes its writes and post-emit verifies pass.
forbidsstopping while .prd has items. stopping while git is dirty or unpushed. Treating a green local test as substitute for green CI. Marker-files-as-completion ("I created done.txt" is not done).
why it existsThe chair is not done until the seat is empty AND the work is shared with the world. git push is part of "done."

update-docs

README / AGENTS.md / docs refresher UPDATE-DOCS (terminal)
roleUpdate README.md, AGENTS.md, docs/index.html (and any other reference surface) to reflect what changed this session. Commit and push the doc updates. Declare COMPLETE.
fires whenVERIFY passes and .gm/prd.yml is empty.
forbidsUpdating docs that lie. Adding changelog history to AGENTS.md (it lives in git log and CHANGELOG.md). Inventing facts the codebase doesn't support.
why it existsIf the docs drift, the next agent in the next session learns the wrong shape. update-docs is the cushion's gift to your future self.

governance

route discovery + emission legitimacy reference PLAN / EXECUTE / EMIT / VERIFY (loaded once)
roleEncodes the 7 route families (grounding, reasoning, state, execution, observability, boundary, representation), the 16-failure taxonomy, the 4 state planes, the ΔS/λ/ε/Coverage metrics, and the 8-case stress suite.
fires whenLoaded once at session start as a reference. Consulted by PLAN (route tagging), EXECUTE (weak-prior bridge), EMIT (legitimacy gate), VERIFY (stress suite).
forbidsEmission without route_fit examined. Crossing the remote-execution boundary (git push, deploy, message-send) without authorization tier upgrade.
why it existsPaper III's contribution: separates route discovery from emission legitimacy. Without it, "plausible plan" silently authorizes "unearned conclusion."

code-search

mandatory codebase lookup workflow any
roleWraps exec:codesearch. Two-word query → change/add one word per pass → minimum four attempts before declaring absent. PDFs included.
fires whenAny time the agent needs to find anything in the codebase.
forbidsfind, grep, rg, Glob, Explore. All hook-blocked. There is no alternative path.
why it existsNative search is fast. Native search is also a tar pit — agents emit huge regexes, get drowned, and lose their place. The constrained API is faster because it's narrow.

browser

live page witness EXECUTE / EMIT / VERIFY
roleWraps exec:browser (playwriter under the hood). Boots the real surface, navigates, polls for the affected global, asserts the invariant via page.evaluate. Captures witnessed numbers.
fires whenAny edit to code that runs in a browser — client/, docs/, *.html, shaders, page-bundle imports, served JS/CSS, anything visible in the DOM/canvas/WebGL.
forbidsscreenshot without page.evaluate. "node test.js passes" as substitute. Deferring witness to a later phase. Silent skip on actual behavior change.
why it existsThe DOM is the truth. A test that doesn't open the page can't confirm the page works. Browser witness is the cushion's own seat-belt.

ssh

remote shell witness EXECUTE
roleWraps exec:ssh. Reads target hosts from ~/.claude/ssh-targets.json. Runs witnessed commands on remote machines for deploy, monitor, and remote-control workflows.
fires whenAnything that has to happen on a not-this machine.
forbidsRunning remote without an explicit target name. Treating remote success as local witness.
why it existsRemote operations are the most common source of forced closure ("I can't reach that machine from here"). With ssh-as-skill, the agent can reach it from here.

pages

GitHub Pages site scaffolder EXECUTE / EMIT
roleBuildless in-browser site (webjsx + rippleui via CDN), flatspace content aggregation built during GH Actions. The site you're reading is built by this skill.
fires whenCreating or updating a GH Pages site for any project.
forbidsBundling. Build steps the user has to run manually. Static-site generators with their own DSLs.
why it existsMost "static" sites are static + a build pipeline + a generator + an asset bundler. pages is actually static, in the browser, with the smallest possible runtime.

create-lang-plugin

CLI/runtime adapter generator EXECUTE / EMIT
roleWires any CLI tool or language runtime into gm-cc as a lang/ plugin. Adds exec:<id> dispatch, optional LSP diagnostics, optional prompt-context injection. Zero hook configuration required from the user.
fires whenYou want to add a new exec: verb (e.g. exec:zig, exec:elixir, exec:custom-cli).
forbidsAdding runtimes by manually editing the dispatcher. Bypassing the plugin convention.
why it existsExtension should be a one-skill operation, not a five-file refactor.

textprocessing sub-agent

semantic routing layer PLAN · EXECUTE · EMIT — wherever meaning is involved
roleAny task whose correctness depends on understanding language — summarize, classify, extract intent, rewrite, translate, semantic dedup, score, label, decide-if-two-texts-mean-the-same — routes here via Agent(subagent_type='gm:textprocessing', model='haiku', ...). One subagent per item, N items in N parallel calls in one message.
fires whenThe agent is about to write a keyword list, regex, or hand-rolled scoring loop to answer a semantic question. That code path is the stub; the haiku call is the real surface.
forbidsbatching multiple items into one call (dilutes signal). keyword-list semantics dressed as deterministic logic. Code path that "looks like classification" but never asks the model.
why it existsCode does mechanics well and meaning badly. A keyword-list loop deciding semantic questions ships a green check that lies — the value terminates and looks right but is unrelated to what the user asked. The split between mechanics and meaning is the boundary this skill enforces.

memorize sub-agent

background memory writer always — every turn
roleClassifies a fact and writes it to AGENTS.md (top-level rules, gm-the-repo scope) and/or rs-learn (everything classified). Background, non-blocking, parallel.
fires whenAny unknown→known transition the same turn it resolves. Triggers: exec output answers a prior unknown, code read confirms/refutes an assumption, CI log reveals a root cause, user states a preference, fix worked for a non-obvious reason, env quirk observed.
forbidsbatched memorize at end-of-turn. serializing parallel spawns. Saying "I'll memorize this" without firing the Agent call. Saying it ≠ doing it.
why it existsWithout same-turn memorize, every fact dies on context compaction. Without it, every session re-discovers what the previous session already learned. Memory is the difference between an agent and a goldfish with a keyboard.

Glootie's notes on the skill chain

"The chain is thirteen skills because the work has thirteen shapes. You will be tempted to merge planning into execute, or emit into complete, or browser into execute, or textprocessing into something that calls a regex. Don't. Each skill is a different posture on the cushion. Merging them is how you end up standing again." — Glootie, on cushion #7
